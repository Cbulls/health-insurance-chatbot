# 프로젝트 실행 가이드

건강보험 설명 챗봇(`health-insurance-chatbot`, 내부 코드명 **harag**)을
로컬에서 띄우고 PDF를 올려 질의하는 방법을 정리한다.

> 한 줄 요약: `.env` 준비 → `docker compose up --build` → 브라우저에서
> [http://localhost:8000](http://localhost:8000) → PDF 업로드 → 질문.

관련 문서:

- 프로젝트 개요: [README.md](../README.md)

- 창구 사용: [COUNTER_UX.md](COUNTER_UX.md)
- 제품 백로그: [COUNTER_PRODUCT_BACKLOG.md](COUNTER_PRODUCT_BACKLOG.md)

- 남은 작업: [REMAINING_WORK.md](../REMAINING_WORK.md)

## 1. 무엇이 뜨는가

기동하면 하나의 주소(`http://localhost:8000`)에서 다음이 함께 제공된다.

| 구성 | 역할 |
|------|------|
| FastAPI API | 문서 업로드·인덱싱·질의(SSE 스트리밍) |
| 정적 프론트엔드 | PDF 드래그&드롭 업로드 + 채팅 UI (`frontend/`) |
| Qdrant | 벡터 검색(Compose 시 컨테이너, 또는 Cloud/인메모리) |
| 문서 등록부 | 기본 SQLite(`./data/harag.db`), 선택적으로 PostgreSQL |

흐름:

```
브라우저 ─▶ FastAPI (:8000)
             ├─ POST /v1/documents     → 파싱 → 청킹 → 임베딩 → Qdrant
             └─ POST /v1/query/stream  → 검색 → LLM → SSE(token/citations/abstain)
```

**API 키 없이도 데모 가능**하다. 임베딩/LLM 키가 없으면 로컬 폴백
(결정적 해시 임베딩 + 추출형 답변)으로 동작한다. 운영 품질이 필요하면
`.env`에 키를 넣는다.

## 2. 사전 요구사항

### 공통

- Git으로 이 저장소를 클론한 상태
- 포트 **8000**(API/UI), Compose 사용 시 **6333**(로컬 Qdrant) 사용 가능

### 방법 A — Docker (권장)

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) 또는 Docker Engine + Compose v2
- (선택) LLM/임베딩 API 키, Qdrant Cloud 계정

### 방법 B — 로컬 Python

- **Python 3.11 이상** (Docker 이미지는 3.12)
- `pip` / `venv`
- (선택) 로컬 Qdrant 또는 Qdrant Cloud. `QDRANT_URL`을 비우면 **인메모리** Qdrant로 동작(프로세스 종료 시 벡터 소실)

### 문서(PDF) 주의

- **텍스트가 추출되는 PDF**만 인덱싱된다.
- 스캔(이미지) PDF는 OCR이 없어 `failed`가 된다.

## 3. 환경변수(`.env`) 준비

```bash
cd health-insurance-chatbot
cp .env.example .env
```

`.env`는 git에 올리지 않는다. 키를 채우지 않아도 서버는 기동된다(로컬 폴백).

로컬에서 `uvicorn`을 직접 실행해도 저장소 루트의 `.env`를 자동으로 읽는다
(`src/harag/config/settings.py`). Docker Compose는 Compose가 `.env`를
서비스 환경변수로 주입한다.

### 3.1 키 없이 데모 (최소 설정)

`.env`를 비워 두거나 예시 그대로 두면 된다.

- 임베딩: 로컬 해시 폴백
- LLM: 로컬 추출형 폴백
- Qdrant: Compose면 `http://qdrant:6333`, 로컬 uvicorn이면 인메모리(URL 비움)

품질은 낮지만 업로드→검색→답변 배관을 확인할 수 있다.

### 3.2 Gemini (Google AI Studio) — 운영 품질 예시

[Google AI Studio](https://aistudio.google.com/)에서 API 키를 발급한다.

```dotenv
LLM_API_BASE=https://generativelanguage.googleapis.com/v1beta/openai
LLM_API_KEY=여기에_키
LLM_MODEL=gemini-flash-latest
LLM_REWRITE_MODEL=gemini-flash-lite-latest

EMBEDDING_API_BASE=https://generativelanguage.googleapis.com/v1beta/openai
EMBEDDING_API_KEY=여기에_같은_키
EMBEDDING_MODEL=gemini-embedding-001
EMBEDDING_DIM=768
EMBEDDING_SEND_DIMENSIONS=true

RATE_LIMIT_QPM=5
LLM_MAX_COST_PER_QUERY_USD=0.05
LLM_MAX_RETRIES=1
```

무료 등급은 RPM이 낮다. 질문당 최대 3콜(rewrite+embed+answer)이므로
`RATE_LIMIT_QPM`은 Gemini RPM÷3 이하(기본 5)로 둔다.
과금·한도 정책은 [GEMINI_COST_STRATEGY.md](GEMINI_COST_STRATEGY.md),
일일 예산은 `python scripts/gemini_quota_budget.py`로 산출한다.
MVP에서는 AI Studio **Billing을 연결하지 않는다.**

### 3.3 OpenAI 호환 (OpenAI / Upstage 등)

```dotenv
LLM_API_BASE=https://api.openai.com/v1
LLM_API_KEY=...
LLM_MODEL=gpt-4o-mini

EMBEDDING_API_BASE=https://api.openai.com/v1
EMBEDDING_API_KEY=...
EMBEDDING_MODEL=text-embedding-3-small
EMBEDDING_DIM=1536
# 차원 축소가 필요하면(예: 512):
# EMBEDDING_DIM=512
# EMBEDDING_SEND_DIMENSIONS=true
```

Upstage 예: `LLM_API_BASE=https://api.upstage.ai/v1`, `LLM_MODEL=solar-pro`.

### 3.4 Qdrant

| 상황 | 설정 |
|------|------|
| `docker compose` 기본 | Compose가 `QDRANT_URL=http://qdrant:6333`을 넣음. `.env`의 URL을 비우거나 덮어쓰지 않으면 로컬 컨테이너 사용 |
| Qdrant Cloud | `.env`에 클러스터 URL + API 키 (아래) |
| 로컬 uvicorn만, URL 비움 | 인메모리(`:memory:`). 재시작 시 인덱스 소실 |

Qdrant Cloud 예:

```dotenv
QDRANT_URL=https://<cluster-id>.<region>.cloud.qdrant.io:6333
QDRANT_API_KEY=...
QDRANT_COLLECTION=harag_pdf_mvp
QDRANT_DISK_BUDGET_MB=3072
```

**임베딩 모델·차원(`EMBEDDING_DIM`)을 바꾸면** 기존 컬렉션과 차원이 맞지 않는다.
컬렉션을 삭제하거나 `QDRANT_COLLECTION`을 새 이름으로 바꾼 뒤 PDF를 다시 올린다.

#### Qdrant 저장·호출 효율 (무료 티어 권장)

디스크·RAM이 빠듯한 Cloud 무료 사양에서는 아래를 권장한다.

```dotenv
EMBEDDING_DIM=512
EMBEDDING_SEND_DIMENSIONS=true
QDRANT_COLLECTION=harag_pdf_512   # 차원 바꾼 뒤 새 이름
QDRANT_DISK_BUDGET_MB=3072
QDRANT_PAYLOAD_BYTES_PER_POINT=1536
HYBRID_PREFETCH_MULT=2
QDRANT_UPSERT_BATCH_SIZE=64
```

앱 동작 요약:

- **payload v2**: 포인트에 슬림 필드만 저장(구버전 `meta` dump는 읽기 호환)
- **재적재 GC**: 같은 문서 재인덱싱 시 기존 포인트를 지운 뒤 upsert(고아 청크 제거)
- **용량 가드**: sparse·세그먼트 보정 공식 + approximate count 캐시
- `/health`의 `capacity`에 `per_point_estimate`, `payload_schema`, `approx` 포함

### 3.5 자주 쓰는 기타 변수

| 변수 | 기본/의미 |
|------|-----------|
| `TOP_K` | 검색 상위 k (기본 20) |
| `MIN_SCORE` | 최소 유사도 (기본 0.15) |
| `MAX_UPLOAD_BYTES` | 업로드 크기 상한 (기본 100MB) |
| `RATE_LIMIT_QPM` | owner당 분당 요청 상한(기본 5). `0`이면 비활성 |
| `LLM_MAX_COST_PER_QUERY_USD` | 답변 LLM 예상비용 상한(기본 0.05) |
| `LLM_MAX_RETRIES` | 429 재시도 횟수(기본 1, 무료 티어) |
| `LLM_REWRITE_MAX_CHARS` | 재작성 프롬프트 문자 상한(기본 2000) |
| `ALLOWED_ORIGINS` | CORS. 비우면 교차 오리진 차단(프론트는 같은 오리진) |
| `AUTH_JWT_SECRET` | 설정 시 Bearer JWT 강제(fail-closed). 비우면 `X-Owner-Id` 헤더 모드 |
| `AUTH_OIDC_MOCK` | `true`면 파일럿용 모의 SSO (`/v1/auth/login`) |
| `DATABASE_URL` | 기본 `sqlite:///./data/harag.db` |
| `LLM_REWRITE_ENABLED` | 멀티턴 질의 재작성 on/off |

전체 목록과 주석은 [.env.example](../.env.example)을 본다.

### 3.6 기관 계정 로그인 (파일럿 SSO)

실 IdP 없이 **「기관 계정으로 로그인」** 버튼을 쓰려면 모의 SSO를 켠다.
공용 규정 업로드·부서 ACL을 같이 검증할 때 이 모드를 권장한다.

**1) `.env`에 추가**

```dotenv
# 32바이트 이상 권장. 예시는 로컬 전용 — 운영 시크릿으로 쓰지 말 것
AUTH_JWT_SECRET=local-dev-jwt-secret-change-me-32b
AUTH_OIDC_MOCK=true
# 모의 로그인 후에는 Bearer가 필요하므로 데모 owner 헤더는 꺼도 됨(선택)
# AUTH_ALLOW_DEMO_OWNER=false
```

**2) API 재기동**

```bash
# Compose
docker compose up --build

# 또는 로컬 uvicorn
uvicorn harag.api.main:app --app-dir src --reload --port 8000
```

**3) 브라우저에서 로그인**

1. [http://localhost:8000](http://localhost:8000) 접속  
2. 왼쪽 **접속** 패널에서 **기관 계정으로 로그인** 클릭  
3. `/v1/auth/login` → 모의 IdP가 JWT를 발급 → URL 해시로 돌아와 `sessionStorage`에 저장  
4. 상태 문구에 `로그인: pilot-admin · claims · 공용 등록 가능` 비슷한 표시가 나오면 성공  

모의 로그인 기본 클레임:

| 클레임 | 기본값 | 의미 |
|--------|--------|------|
| `sub` | `pilot-admin` | 사용자 ID |
| `dept` | `claims` | 부서 — 공용 규정 공유 범위 |
| `roles` | `doc_admin` | 공용 규정 등록·삭제 가능 |

쿼리로 바꿀 수 있다 (북마크/테스트용):

```
http://localhost:8000/v1/auth/login
# 내부적으로 mock-login?sub=...&dept=...&roles=...
# 직접 호출 예:
# /v1/auth/mock-login?state=<login이_준_state>&sub=alice&dept=claims&roles=staff
```

`roles=staff`(admin/doc_admin 없음)면 업로드 대상에 **공용·라이브러리**가 안 보이고, 같은 `dept`의 공용 문서와 **사내 지식 라이브러리**는 조회·질의에 포함된다.

**4) 사내 지식 라이브러리 + 약관 검증 (권장 시나리오)**

목적: 직원이 올린 보험 약관과, 관리자가 올린 법령을 **한 질의에서 같이 참조**하는지 확인.

1. `doc_admin`으로 로그인 → **컬렉션+** 로 `건강보험법령` 생성  
2. 업로드 대상 = `라이브러리 · 건강보험법령` → 법령 PDF 업로드 → 준비됨  
3. 로그아웃 후 staff로 로그인  
   (`/v1/auth/mock-login?state=...&sub=alice&dept=claims&roles=staff` — login 플로우의 state 사용)  
4. 업로드 대상 = `내 업로드` → 보험사 약관 PDF 업로드  
5. 약관·법령이 겹치는 내용으로 질문 → 출처에 양쪽 문서가 후보로 나올 수 있음  

라이브러리 문서는 모든 사용자 검색 ACL에 `lib:knowledge`로 항상 포함된다.

**5) 공용 규정 올려 보기**

1. `doc_admin`으로 로그인 → 업로드 대상 **공용 규정**  
2. 텍스트 PDF/HWPX 업로드 → **공용 규정** 목록에 표시  
3. staff(같은 `dept`)로 보면 공용 목록에 보이고, 다른 직원의 **내 업로드**는 안 보인다  

**6) 로그아웃**

- **로그아웃** 버튼 → 토큰·목록 초기화  
- 고급 · JWT 붙여넣기는 개발/장애 대응용이다  

**운영 IdP로 바꿀 때**

`AUTH_OIDC_MOCK=false`로 두고 authorize/token/JWKS·redirect URI를 설정한다.
절차는 [SECRETS_OPS.md](SECRETS_OPS.md) 「OIDC SSO redirect 등록」을 본다.
창구·라이브러리 사용은 [COUNTER_UX.md](COUNTER_UX.md)를 본다.

| 증상 | 확인 |
|------|------|
| 로그인 버튼이 안 보임 | `AUTH_OIDC_MOCK=true` + `AUTH_JWT_SECRET` 설정 후 **재기동**. `/v1/auth/config`의 `sso_enabled`가 true인지 확인 |
| 로그인 후 401 | 시크릿이 기동 중에 바뀌었거나 토큰이 만료됨 → 다시 로그인 |
| 공용 체크가 안 보임 | JWT에 `roles`에 `admin` 또는 `doc_admin` 필요(모의 기본값은 `doc_admin`) |

## 4. 실행 방법 A — Docker Compose (권장)

### 4.1 기동

```bash
cd health-insurance-chatbot
cp .env.example .env   # 아직 없다면. 키는 선택
docker compose up --build
```

최초 빌드는 수 분 걸릴 수 있다. 로그에 API/uvicorn이 뜨면 준비된 것이다.

### 4.2 접속

브라우저에서 **http://localhost:8000**

- 왼쪽: 접속(데모 세션 또는 **기관 계정으로 로그인**) · PDF 업로드 · 공용 규정 / 내 업로드
- 오른쪽: 질문 입력 → 스트리밍 답변 + 출처(citations)
- 파일럿 SSO: [§3.6](#36-기관-계정-로그인-파일럿-sso)  
- 데모만: 시크릿을 비우면 브라우저 세션 ID(`X-Owner-Id`)로 본인 문서만 검색

### 4.3 중지·재시작

```bash
# 포그라운드면 Ctrl+C
docker compose down          # 컨테이너 중지(볼륨은 유지)
docker compose up --build    # 다시 기동
```

Qdrant·SQLite 데이터는 Compose 볼륨(`qdrant_storage`, `harag_data`)에 남는다.
완전 초기화가 필요하면 `docker compose down -v` (데이터 삭제).

### 4.4 (선택) PostgreSQL 문서 등록부

```bash
docker compose --profile postgres up -d postgres
```

`.env`에 다음을 넣고 API를 재기동한다.

```dotenv
# Compose 네트워크 안 API 컨테이너 기준 호스트명은 postgres
DATABASE_URL=postgresql+psycopg://harag:harag@postgres:5432/harag
```

이미지에 `psycopg`가 없다면 Dockerfile/`pip`에 `.[postgres]` 반영이 필요할 수 있다.
호스트에서만 API를 돌릴 때는 호스트명을 `localhost`로 둔다.

## 5. 실행 방법 B — 로컬 Python (Docker 없이)

### 5.1 가상환경·설치

```bash
cd health-insurance-chatbot
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -e .
cp .env.example .env               # 필요 시 키·Qdrant URL 편집
```

### 5.2 Qdrant 선택

1. **인메모리(가장 단순)**  
   `.env`에서 `QDRANT_URL=` 비움 → 프로세스 안에서만 동작. 재시작 시 벡터 사라짐.

2. **로컬 Qdrant 컨테이너만**  
   ```bash
   docker run -p 6333:6333 -v qdrant_storage:/qdrant/storage qdrant/qdrant:latest
   ```
   `.env`:
   ```dotenv
   QDRANT_URL=http://127.0.0.1:6333
   ```

3. **Qdrant Cloud**  
   §3.4처럼 URL·API 키 설정.

### 5.3 API 기동

```bash
source .venv/bin/activate
uvicorn harag.api.main:app --host 127.0.0.1 --port 8000 --reload
```

브라우저: **http://127.0.0.1:8000**

`--reload`는 코드 변경 시 자동 재시작(개발용).

### 5.4 헬스체크

```bash
curl -s http://127.0.0.1:8000/health
```

예: `{"status":"ok","capacity":{...}}`

## 6. 첫 사용 절차 (UI)

1. http://localhost:8000 접속
2. 왼쪽에서 **텍스트 PDF** 업로드
3. 문서 상태가 **준비됨**(ready)이 될 때까지 대기(수 초~수십 초)
4. 오른쪽에 문서 내용과 관련된 질문 입력 후 보내기
5. 답변이 토큰 단위로 스트리밍되고, 아래에 출처가 붙는지 확인
6. 문서와 무관한 질문이면 **"모른다"(abstain)** 쪽이 나와야 한다  
   (폴백·임베딩 품질에 따라 기대와 다를 수 있음 → 스모크로 배관 검증)

**새 대화** 버튼은 멀티턴 맥락(`conversation_id`)을 리셋한다.

## 9. API 요약 (직접 호출)

프론트와 동일하게 소유자 헤더를 붙인다.

```bash
OWNER=demo-user-1

# 업로드
curl -s -X POST http://localhost:8000/v1/documents \
  -H "X-Owner-Id: $OWNER" \
  -F "file=@./my.pdf;type=application/pdf"

# 상태 조회
curl -s http://localhost:8000/v1/documents/<document_id> \
  -H "X-Owner-Id: $OWNER"

# 질의 (JSON)
curl -s -X POST http://localhost:8000/v1/query \
  -H "X-Owner-Id: $OWNER" \
  -H "Content-Type: application/json" \
  -d '{"query":"출장비 한도는?"}'

# 질의 (SSE 스트림)
curl -sN -X POST http://localhost:8000/v1/query/stream \
  -H "X-Owner-Id: $OWNER" \
  -H "Content-Type: application/json" \
  -d '{"query":"출장비 한도는?","conversation_id":"c-demo-1"}'
```

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/health` | 헬스·용량 |
| GET | `/v1/auth/config` | SSO·데모 가능 여부 |
| GET | `/v1/auth/login` | 기관 로그인 시작(리다이렉트) |
| GET | `/v1/auth/me` | 현재 사용자·부서·공용/라이브러리 등록 권한 |
| GET/POST/DELETE | `/v1/collections` | 사내 지식 라이브러리 컬렉션 |
| POST | `/v1/documents` | 업로드 (`shared=true` 또는 `collection_id`) → 202 |
| GET | `/v1/documents` | 내 업로드 + 부서 공용 규정 |
| GET | `/v1/documents/{id}` | 상태(`processing` / `ready` / `failed`) |
| DELETE | `/v1/documents/{id}` | 문서·인덱스 삭제(공용은 관리자) |
| POST | `/v1/query` | 동기 JSON 답변 |
| POST | `/v1/query/stream` | SSE (`token` / `citations` / `abstain` / `done` / `error`) |

`AUTH_JWT_SECRET`(또는 OIDC JWKS)을 설정하면 `Authorization: Bearer <jwt>`가 필요하고,
헤더만으로 owner를 스푸핑할 수 없다. 모의 SSO는 [§3.6](#36-기관-계정-로그인-파일럿-sso).

## 11. 문제 해결

| 증상 | 확인 |
|------|------|
| `localhost:8000` 연결 거부 | 컨테이너/`uvicorn`이 떠 있는지, 포트 충돌 |
| 업로드 후 영원히 처리중 | API 로그·Qdrant 연결(`QDRANT_URL`). Cloud면 키·URL·방화벽 |
| `capacity_exceeded` | Qdrant 디스크 예산. 문서 삭제 또는 `QDRANT_DISK_BUDGET_MB` / 차원 축소 |
| 답이 항상 부실·이상 | 키가 비어 로컬 폴백인지 확인. Gemini면 `LLM_API_BASE`가 AI Studio OpenAI 호환 URL인지 |
| `llm_unavailable` / 429 | 프로바이더 쿼터·`RATE_LIMIT_QPM`. Gemini면 [GEMINI_COST_STRATEGY.md](GEMINI_COST_STRATEGY.md) |
| 임베딩 차원 오류 | `EMBEDDING_DIM` 변경 후 예전 컬렉션 사용 중 → 컬렉션 삭제/이름 변경 후 재업로드 |
| 스캔 PDF `failed` | OCR 미구현. 텍스트 PDF 사용 |
| JWT 설정 후 401 | `AUTH_JWT_SECRET` 사용 중이면 유효 Bearer 필요. 데모만 쓰려면 시크릿을 비울 것. 파일럿은 §3.6 모의 로그인 |
| 기관 로그인 버튼 없음 | `AUTH_OIDC_MOCK=true` + `AUTH_JWT_SECRET` 후 재기동. `GET /v1/auth/config` |
| CORS 오류 | 프론트는 같은 오리진 서빙이 기본. 다른 오리진에서 호출 시 `ALLOWED_ORIGINS` |
| Compose에서 Cloud Qdrant | `.env`의 `QDRANT_URL`/`QDRANT_API_KEY`가 api 서비스에 전달되는지 확인(로컬 `http://qdrant:6333`을 덮어씀) |

로그 보기 (Compose):

```bash
docker compose logs -f api
```

## 12. 리랭커 (Cross-Encoder, 선택)

기본은 **어절 겹침 Lexical** 폴백이다. 운영 정밀도를 쓰려면 TEI/vLLM 호환
리랭커 서버를 띄우고 `.env`에 URL을 넣는다.

```dotenv
RERANK_ENABLED=true
RERANKER_SERVER_URL=http://127.0.0.1:8080
RERANK_TOP_N=5
# HTTP CE면 기본 blend=0, RERANK_MIN_SCORE=MIN_SCORE
```

기동 로그 예:

- `rerank=lexical(blend=0.70,...)` — URL 없음
- `rerank=http(blend=0.00,...)` — TEI 연결

진단·고도화 배경: [RERANKER_CTO_REVIEW.md](./RERANKER_CTO_REVIEW.md).

정확도 상한은 리랭커보다 **실임베딩·하이브리드 컬렉션·멀티턴 재작성**이 먼저다.
`embedding=local(hash)` 경고가 뜨면 운영 전에 API 키를 넣는다.
